# 好AI - 安全设计方案

## 一、API Key安全

### 1.1 加密存储

所有API Key使用 **AES-256-GCM** 加密存储：

```
加密流程:
API Key明文 → AES-256-GCM加密 → 密文(iv:authTag:ciphertext) → 存储到数据库

解密流程:
数据库密文 → AES-256-GCM解密 → API Key明文 → 用于调用供应商API
```

**密钥保护**:
- 加密密钥通过环境变量 `API_KEY_ENCRYPTION_KEY` 配置
- 生产环境必须使用强密码（32字节随机字符串）
- 即使数据库被拖库，没有加密密钥也无法还原API Key

### 1.2 密钥使用限制

- API Key仅在后端内部使用，不暴露给前端
- 调用供应商API时使用代理模式，用户无法直接接触密钥
- 所有API Key操作记录日志

## 二、计费安全

### 2.1 费率控制

- **固定倍率**: 所有API按 `成本价 × 1.2` 销售
- **向上取整**: 费用计算向上取整到4位小数，确保利润
- **最低消费**: 每次调用至少扣除 0.0001 元

### 2.2 余额保护

- **余额不足即时停用**: 余额≤0时立即返回402，拒绝服务
- **原子性扣款**: 使用数据库事务防止并发导致负余额
- **双重余额检查**: 扣款前和扣款时双重验证余额

### 2.3 消费透明

每次API调用返回详细计费信息：
```json
{
  "success": true,
  "usage": {
    "input_tokens": 100,
    "output_tokens": 200,
    "cost": 0.0245,
    "model": "GPT-5",
    "provider": "OpenAI"
  },
  "balance": 99.9755,
  "alert_message": null
}
```

## 三、防攻击措施

### 3.1 速率限制（Rate Limiting）

| 限制项 | 限制值 |
|--------|--------|
| 每分钟请求数 | 30次 |
| 超出封锁时间 | 5分钟 |
| 统计窗口 | 1分钟 |

超过限制返回：
```json
{
  "success": false,
  "error": "请求过于频繁，请60秒后再试",
  "code": "RATE_LIMITED",
  "retryAfter": 60
}
```

### 3.2 并发保护

使用数据库事务保证余额操作的原子性：
```sql
UPDATE users
SET balance = balance - ?
WHERE id = ? AND balance >= ?
```

如果余额不足，`changes` 返回0，扣款失败。

### 3.3 CSRF保护

- Cookie设置 `SameSite=Lax`
- 关键操作需要验证用户身份

## 四、支付安全

### 4.1 微信支付签名验证

回调必须验证签名：
```typescript
verifyWechatPaySignature(params, signature, apiKey)
```

验证流程：
1. 提取所有参数（排除sign字段）
2. 按字典序排序
3. 拼接成 `key=value&...` 格式
4. 末尾拼接API密钥
5. MD5加密后与传入签名比对

### 4.2 支付宝签名验证

使用RSA2公钥验证：
```typescript
verifyAlipaySignature(params, signature, alipayPublicKey)
```

### 4.3 支付状态机

```
订单创建 → pending
    ↓
支付回调验证 → completed 或 failed
    ↓
余额更新（仅completed）
```

### 4.4 防重复回调

同一订单号只处理一次，重复回调直接返回成功。

## 五、用户数据保护

### 5.1 敏感信息处理

| 信息类型 | 处理方式 |
|----------|----------|
| API Key | AES-256-GCM加密，不返回前端 |
| 用户密码 | 加密存储（微信/支付宝授权） |
| 手机号 | 脱敏显示（138****8888） |

### 5.2 日志脱敏

消费记录日志不记录敏感内容，仅记录：
- 用户ID
- API配置ID
- Token数量
- 费用金额
- 时间戳

## 七、应急响应

### 7.1 发现API Key泄露

1. 立即在管理后台停用该API配置
2. 联系供应商更换API Key
3. 检查消费记录，确认损失
4. 更新API配置

### 7.2 发现异常消费

1. 封停相关用户账号
2. 保存日志证据
3. 联系用户核实
4. 必要时报警处理

## 八、技术选型

| 安全功能 | 实现方案 |
|----------|----------|
| 密钥加密 | AES-256-GCM（battle-tested） |
| 签名算法 | MD5（微信）、RSA2（支付宝） |
| 速率限制 | 内存Map（生产环境用Redis） |
| 事务保护 | SQLite WAL + 事务 |
